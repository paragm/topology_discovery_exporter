Topology Discovery Exporter

Prometheus exporter that discovers network topology via SNMP/LLDP and enriches Prometheus target files with topology labels.

Prometheus SNMP / LLDP Go :10042

Overview

  • Exposes discovery metrics — run status, switch/host/link counts, per-switch errors
  • Enriches Prometheus targets — adds switch_id, switch_port, uplink_switch_l1, network_path labels to file_sd target files
  • Persists topology — stores discovered topology in SQLite for history and change detection
  • Triggers Prometheus reload — automatically reloads Prometheus when target files change
  • Periodically polls network switches — via snmp_exporter, parses LLDP neighbor data, builds a topology graph

Architecture

The exporter sits between your network switches and Prometheus, using snmp_exporter as an SNMP proxy to gather LLDP neighbor tables and build a complete topology graph.

┌─────────────┐ SNMP ┌───────────────┐ HTTP ┌──────────────┐ │ Switches │◄──────────────│ snmp_exporter │◄─────────────│ topology │ │ (LLDP data) │ └───────────────┘ │ _discovery │ └─────────────┘ │ _exporter │ │ │ ┌───────────────┐ file_sd write │ │ │ Prometheus │◄───────────────│ │ │ (targets/) │ POST reload │ │ └───────────────┘ └──────────────┘

Features

  • SNMP/LLDP Discovery — Polls switches through snmp_exporter, parsing LLDP MIB data to identify neighbors, ports, and connections
  • Topology Graph Construction — Builds a directed graph of switch-to-switch and host-to-switch connections, computing network paths and uplink chains
  • Target File Enrichment — Reads Prometheus file_sd target files and injects topology labels (switch_id, switch_port, uplink_switch_l1, network_path)
  • SQLite Persistence — Stores discovered topology in a local SQLite database for historical queries and change detection across restarts
  • Automatic Prometheus Reload — Posts to the Prometheus /-/reload endpoint when enriched target files are written, ensuring label changes take effect immediately
  • Per-Switch Error Tracking — Records and exposes SNMP errors per switch so you can identify unreachable or misconfigured devices

Installation

From Source

git clone https://github.com/paragm/topology_discovery_exporter.git
cd topology_discovery_exporter
make build
sudo make install

Systemd Service

sudo cp examples/systemd/topology_discovery_exporter.service /etc/systemd/system/
sudo cp examples/systemd/topology_discovery_exporter.env /etc/default/topology_discovery_exporter
sudo cp examples/config.yml.example /opt/topology/config.yml
sudo systemctl daemon-reload
sudo systemctl enable --now topology_discovery_exporter

YAML Configuration File

config.yml 
snmp_exporter_url: http://127.0.0.1:9116
snmp_module: lldp_topology
snmp_auth: my_snmp_profile

switches:
  - name: core-sw01
    address: 192.168.0.1
    role: core
  - name: access-sw01
    address: 192.168.0.10
    role: access

prometheus:
  reload_url: http://127.0.0.1:9090/-/reload
  targets_dir: /etc/prometheus/targets

base_target_files:
  - /etc/prometheus/targets/nodes.yml

db_path: /var/lib/topology/topology.db
cache_dir: /var/lib/topology/cache
